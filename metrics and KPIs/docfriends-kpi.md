### Organizational Restructuring (Development Methodology)

#### Organization Structure

Transition from single-team structure (BE-FE-PD-CE-FT silos) to **Tribe or Squad-based** formation.

**CEO-led Model:**
- Risk: "Core executive overload or bottlenecks can collapse the entire team structure"
- Mitigation: "Reduce single authority through organizational duplication"

**CTO-led Model:**
- Should be adopted at this stage (while minimizing in-team duplication and formal authority consolidation, enabling horizontal positioning)
- **FE-Design:** Fully establish the design system so anyone can contribute
- **CE-CX:** Process standardization enabling anyone to substitute
- **BE:** Replaceable if DB standards and comfort are fully understood (though not achievable within this period)
- **BD-PM:** Business-project alignment — minimizing one-directional resource allocation and routing modernization is the greater challenge

**Conclusion:** "Assigning ownership for every process and individual project remains difficult. Regular cadence is needed to reduce business-purpose fragmentation and technical coupling. Each person participates as a project team member and project PM. Transitioning to an AI-First culture requires gradual adoption — immediate 100% AI-First conversion in 1Q with full participation is not feasible."

#### Execution Framework

- Separate PM (PO) authority from project (CE team), formalize execution processes, match and assign personnel
- Assess each role's workload and define AI utilization ratios in development processes
- Match 2–4 personnel per project unit

#### AI Native Project Transition (based on 2024.12 data)

**Service Mastering:**
- Doctalk Reservation (Naver bot, app launch)
- Doctalk Expert QnA (Naver Knowledge iN FAQ, app solution)
- Doctalk CRM Integration (ad analytics service)

**Technical Mastering:**
- Internal/external domain mastering
- Pipeline API mastering & AI resource mastering
- Standardized API grammar construction
- Legacy → AI Native conversion: effort estimation & prioritization
- Legacy code AI Native migration

#### Performance & Compensation

- Introduce Tribe/Squad-level goal-based incentives
- **Feedback Loop (CEO + HR + Unit Team):** Quarterly retrospective reviews (4x per year) / Leader-club mapping feedback
- Financial status reporting intervals (1–2x per year)
- **Performance Rewards (on KPI & BEP achievement):** Team-wide base compensation / On achievement: rewards based on "achievement %" and contribution-weighted lead team selection / On miss: no bonus
